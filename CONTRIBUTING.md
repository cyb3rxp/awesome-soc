# Contributing to Awesome SOC

Thank you for your interest in contributing to Awesome SOC.

This repository is a curated knowledge base of resources related to building, operating, and improving a Security Operations Center (SOC).

Contributions are welcome and appreciated.

---

# How to Contribute

You can contribute by:

- Adding new resources
- Improving descriptions
- Fixing broken links
- Improving structure or formatting
- Suggesting new sections

Please open a Pull Request (PR) with your proposed changes.

---

# Contribution Guidelines

To keep the list useful and high-quality, please follow these rules.

## 1. Quality resources only

Resources should be:

- Relevant to SOC operations
- Technically valuable
- Maintained or widely used
- Not purely promotional

Avoid:

- Marketing pages
- Low-quality blog spam
- Affiliate links

---

## 2. Add a short description

Each resource must include a short description explaining why it is useful.

Example:

Good:

- Sigma — Generic detection rule format for SIEM systems.

Bad:

- Sigma

---

## 3. Place resources in the correct section

Before adding a new section, check if the resource fits in an existing one.

If you believe a new category is needed, explain it in the PR.

---

## 4. Keep descriptions concise

Descriptions should be one sentence.

Example:

Correct:

- Velociraptor — Endpoint visibility and DFIR collection tool.

Avoid long explanations.

---

## 5. Alphabetical order

When possible, add resources in alphabetical order within a section.

This keeps the list easy to maintain.

---

# Pull Request Process

1. Fork the repository
2. Create a new branch
3. Add your changes
4. Open a Pull Request
5. Explain briefly what you added or changed

Example PR title:

Add detection engineering resource: Sigma

---

# Code of Conduct

Please be respectful and constructive in discussions.

---

# Thank You

Your contributions help make this repository a useful reference for the SOC and blue team community.
